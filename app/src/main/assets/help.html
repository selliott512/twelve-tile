Overview

Twelve Tile Puzzle is heavily inspired by the "M12 puzzle" that can be found on the Scientific American website. The goal is to place the 12 tiles at the top of the screen in order by pressing the first two of the three move buttons that appear on the left side of the screen in the correct sequence. What the two move buttons do is controlled by which of the three puzzle types ("M12", "Bubble" and "Custom") is selected.

Prior to this puzzle the M12 puzzle type had been implemented as a Java applet, which can be found on the author's website.

Some of the six buttons are self explanatory, so you should just try them. See the buttons section for a detailed explanation of the buttons.

The Twelve Tile Puzzle is open source. See the license section for details.

Puzzle Types

  • M12 is quite difficult for reasons that have do with group theory. The M12 puzzle is the one puzzle type that this puzzle knows how to solve since the limited number of permutations makes it possible to include a comprehensive database. See the Scientific American website for a discussion behind the M12 puzzle type. For the M12 puzzle type the "Flip" button reverses the order of the tiles and the "Merge" button folds the tiles back onto themselves. This is the default puzzle type.
  • Bubble is named since after the fact that it implements bubble sort. The only important difference between this puzzle type and the bubble sort algorithm is that for this puzzle type the place where items are swapped is fixed and the tiles move.
  • Custom is used to control exactly what first two move buttons do. For all puzzle types each of the two moves can be described by what they do to a solved puzzle. For example, for the M12 puzzle type the the "Flip" button takes a solved puzzle, which can be represented by ABCDEFGHIJKL, and flips it so that the order of the tiles is reversed, which is represented by LKJIHFEDCBA. Similarly for the custom puzzle type the two moves buttons my be configured by the "Custom Move 1 Result" and "Custom Move 2 Result" preferences on the preferences page (the page that is navigated to by selecting "Preferences" when the "Menu" button is pressed). Optionally the label that appears on the first two move buttons can be controlled by the "Custom Move 1 Name" and "Custom Move 2 Name" preferences on the preferences page.

Strategy

The M12 puzzle type is quite difficult. Although a comprehensive solution is beyond the scope of this document consider the following:

  • Merge changes everything other than the first tile. So it's possible to get the first two tiles by pressing "Merge" until A is in the last location. Then press flip to get A into the first location and finally continue to press "Merge" until B is in the second location.
  • Using the "Cheat" preference that can be found in the preferences page illustrates which move gets the puzzle closer to solved.
  • For the M12 puzzle type knowing the location of any five tiles is sufficient to deduce the location of all 12 tiles. In order to assist with focusing on five of the tiles try turning on the "Five" preference in the preferences page.

The Bubble puzzle type is easier than the M12 puzzle type. It's solved by applying the bubble sort algorithm. Since the "Swap" button causes the first two tiles to exchange places press "Swap" if the first tile comes after the second one and press "Roll" otherwise. One caveat is that one of the tiles should not be moved. For example, never swap tile A with another tile.

Buttons

  • Moves

    • Move 1 is the first of the three move buttons. It is is labeled "Flip" by default, but it's label changes as a function of the puzzle type.
    • Move 2 is the second of the three move buttons. It is is labeled "Merge" by default, but it's label changes as a function of the puzzle type.
    • Undo undoes the previous move.

  • Actions

    • Random picks a random permutation that is valid for the current puzzle type. Internally this is done by doing 100 random moves.
    • Reset restores the puzzle to it's solved state.
    • Rewind undoes all moves since the last time the puzzle was randomized by clicking the "Random" button. Use this to work on a particular random puzzle by quickly reverting to the same random puzzle.
